- 소프트웨어 보안 취약점을 분류한 CWE(Common Weakness Enumeration)를 소개한다.

- CWE는 보안에 영향을 미치는 소프트웨어 및 하드웨어의 약점(weaknesses)과 취약점(vulnerabilites) 유형을 정리한 목록이다. 
- CWE는 600개가 넘는 카테고리로 구성되어 있으며 이 중 대표적인 C, C++, JAVA의 취약점을 다루는 카테고리는 다음과 같다.
CWE-658: 총 80개의 C언어 약점으로 구성된다.<br>
CWE-659: C++<br>
CWE-660: 자바

## 약점(weaknesses)과 취약점(vulnerabilites) 구분
- 약점(weaknesses)과 취약점(vulnerabilites)은 엄밀히 다른 의미를 가진다.  
- 약점(weaknesses)은 '공격에 활용될 여지가 있는 오류'를 의미하고 취약점(vulnerabilites)은 '실제로 공격 구현이 가능한 오류'를 의미한다. 이론상 존재하는 위험 요소는 약점(weaknesses)이며, 실제적으로 공격이 가능한 것을 취약점(vulnerabilites)라고 한다. 
- 소프트웨어 약점(weaknesses)은 취약점(vulnerabilites)으로 이어질 수 있다. 즉, 약점은 취약점 보다 더 넓은 의미를 가진다. 
- Buffer Overflow, Double Free와 같은 오류가 약점(weaknesses)에 해당된다.

## CWE 구조(Nature)
- View를 제외한 다른 약점들은 다른 유형의 약점들과 1:N 관계를 가지는 그래프로 구성된다. 이러한 그래프 구조를 통해 약점들을 상세한 단계로 분류할 수 있다.

![image](/assets/images/2020-07-03-CWE658-1/image1.png)

### CWE Nature
- MemberOf -> HasMember
- CanPrecede -> CanFollow
- StartsChain -> StartsWith
- ParentOf -> ChildOf
- PeerOf
- CanAlsoBe
- Requires -> RequiredBy
 
- Relationships 섹션의 항목 중 Nature으로 약점 간의 관계를 파악 할 수 있다. 예를 들어 ParentOf(cwe-129) -> ChildOf(cwe-119) 관계는 cwe-129가 부모노드, cwe-119가 자식노드의 관계에 있다는 것을 의미한다. 정리하면 cwe-129가 cwe-119를 포함 할 수 있는 약점이며 cwe-119는 cwe-129의 세분화된 하위 약점으로 볼 수 있다. 이와 같은 Nature 종류를 통해 약점 간의 관계를 파악 할 수 있다.

## CWE 유형(Type)

![image](/assets/images/2020-07-03-CWE658-1/image2.png)

- CWE 유형 중 CWE 658에서 사용되는 유형에 대해여 소개하겠다. 


### Weakness - Class
- 구체적이지 않고 추상적으로 설명되어 있다.<br>
ex) Concurrent Execution using Shared Resource with Improper Synchronization ('Race Condition'): 동시에 실행되는 스레드들이 공유 자원에 접근하려고 할 때 발생하는 현상을 'Race Condition'이라고 한다. Race Condition이 발생하지 않도록 구체적인 설명이 없기에, Weakness - Class에 포함된다. 


### Weakness - Base
추상적으로 설명되어 있지만 구체적인 방법을 유추하기에 세부 정보를 제공한다. Weakness - Class 보다 구체적으로 설명되어 있다.<br>
ex) Missing Default Case in Switch Statement: switch ... case 문에서 default 구문을 추가하여 예상치 못한 상황을 확인해야 한다. 추상적인 설명이지만 구체적인 방법을 유추할 수 있으므로 Weakness - Base에 포함된다. 


### Weakness - Variant
- 특정 프로그래밍 언어 또는 기술에 대한 약점이다. Weakness - Base 보다 더 구체적으로 설명되어 있다.<br>
ex) Double Free: C언어는 개발자가 직접 메모리를 할당하고 해제(free)할 수 있다. free 함수는 C언어에서 사용하기에 Weakness - Variant에 포함된다.


### Compound Element - Composite
- 두 개 이상의 약점이 동시에 존재하는 경우, 잠재적으로 발생하는 취약점으로 이어진다. 약점을 제거하면 위험이 제거되거나 감소한다.


### Compound Element - Named Chain
- 서로 연결될 수 있는 둘 이상의 약점들이다. 하나의 약점은 또 다른 약점으로 이어질 수 있는 필요조건이며, 취약점이 발생할 수 있는 상태로 이어질 수 있다.
